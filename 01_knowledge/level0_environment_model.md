# Level 0 — Environment Model (Process & venv)

## 1. Что такое venv

venv — это директория со своей версией:
- python-интерпретатора
- site-packages
- файла pyvenv.cfg

venv НЕ является процессом.
venv НЕ является контейнером.
Это структура файловой системы.

---

## 2. Что делает активация

Команда:

    source .venv/bin/activate

выполняет shell-скрипт, который:

1. Изменяет PATH (добавляет .venv/bin в начало)
2. Устанавливает переменную VIRTUAL_ENV
3. Меняет PS1 (визуальный индикатор)

Активация меняет состояние ТЕКУЩЕГО shell-процесса.

---

## 3. Модель процессов

Каждый терминал = отдельный shell-процесс.

Когда запускается программа:

    python script.py

происходит:

    fork() → exec()

Дочерний процесс наследует переменные окружения родителя.

Переменные окружения распространяются:
    родитель → ребёнок
но не наоборот.

---

## 4. Новый shell внутри shell

Если внутри активированного venv выполнить:

    bash

создаётся дочерний shell.

Он наследует:
- PATH
- VIRTUAL_ENV

Но может пересобрать PS1, поэтому индикатор (.venv) может исчезнуть.

Вывод:
Визуальный индикатор ≠ реальное состояние среды.

Проверять нужно через:
    which python
    echo $VIRTUAL_ENV

---

## 5. deactivate

deactivate:
- восстанавливает прежний PATH
- удаляет VIRTUAL_ENV

venv при этом никуда не исчезает.
Меняется только окружение текущего shell.

---

## 6. Уровни среды

1. Файловая система (venv как директория)
2. Shell-процесс (переменные окружения)
3. Дочерние процессы (наследование окружения)

Понимание этих уровней даёт контроль над средой.
