🧠 1. Саммари проекта (состояние на сейчас)
📍 Общий уровень

Проект — это:

локальная RAG-система поверх библиотеки книг

Не прототип. Рабочая система с контролем качества.

⚙️ Что реально работает
1. Индексация
сканирование каталога
парсинг:
fb2 / fb2.zip
epub (хорошо)
pdf (минимально, но уже с текстом)
сохраняется:
title / author
raw_text
description
2. Работа с текстом
raw_text сохраняется в БД
режется на chunks
chunks → embeddings
embeddings хранятся в SQLite
3. Retrieval
cosine similarity
keyword boost (ограниченный)
фильтр коротких чанков
top-N
4. RAG (ask2)

Pipeline:

query → embedding → chunks → context → LLM
5. Контроль качества (ключевой апгрейд)

Добавлено:

- score threshold
- definition detection
- fallback на LLM

Pipeline сейчас:

retrieval → rerank (эвристика)
         → quality check:
             - max_score
             - has_definition
         → fallback (если плохо)
📊 Текущее поведение
✔ Хорошо
система не врёт уверенно
fallback работает
определения обрабатываются корректно
мусор из художественных книг почти не проходит
❗ Ограничения
Retrieval всё ещё шумный
Нет выбора “правильной книги”
Только начало книги (ограничение парсинга)
Нет структуры (всё = raw_text)
Один pipeline для всех типов вопросов
🧠 Ключевые инсайты (очень важно)
1. Проблема не в LLM
узкое место = контекст + retrieval
2. Библиотека ≠ интернет
общие вопросы → не надо через RAG
3. Нельзя один pipeline на всё

Нужно разделение:

- definition
- general knowledge
- book-aware queries
4. Главный недостающий элемент
entity resolution (поиск книги по названию)
🚧 Что пробовали (и что не зашло)
жёсткие фильтры → ломают ответы
бусты → нестабильны
чистый semantic search → шум
✅ Что сработало
fallback
threshold
ограничение определения
