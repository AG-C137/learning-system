### Архитектура

`book_indexer` - это распределенная система для индексации и поиска текстов в книгах, использующая Retrieval-Augmented Generation (RAG) подход. Система включает следующие компоненты:

1. **Индексация (Indexing)**: Принимает книги, разбивает их на чанки, вычисляет эмбеддинги и сохраняет их в базе данных.
2. **Хранилище (Storage)**: SQLite база данных для хранения индексов и метаданных.
3. **Поиск (Search)**: Реализует семантический поиск на основе эмбеддингов с использованием косинусной близости.
4. **Ответ generation** (`ask2`): Использует RAG подход для генерации ответа на запрос.

### Основные модули и их взаимосвязи

1. **index (src/book_indexer/index.py)**:
   - Отвечает за загрузку книг, разбиение на чанки и вычисление эмбеддингов.
   - Использует `embedder` для вычисления эмбеддингов.

2. **storage (src/book_indexer/storage.py)**:
   - Отвечает за хранение индексов в SQLite базе данных.

3. **search (src/book_indexer/search.py)**:
   - Реализует семантический поиск на основе эмбеддингов.
   - Использует `semantic_search_and_answer` для получения результатов поиска и их ранжирования.

4. **ask2 (src/book_indexer/cli/main.py)**:
   - Основной интерфейс командной строки, который обрабатывает запросы от пользователей.
   - Использует `semantic_search_and_answer` для генерации ответов на основе запросов.

### Flow данных

1. **Индексация**:
   - Загрузка книг и их разбиение на чанки.
   - Вычисление эмбеддингов для каждого чанка с использованием `embedder`.
   - Сохранение индексов в SQLite базе данных.

2. **Поиск**:
   - Получение запроса от пользователя.
   - Вызов `semantic_search_and_answer` для получения релевантных чанков.
   - Ранжирование результатов поиска.
   - Возврат отсортированных результатов пользователю.

3. **Ответ generation**:
   - Получение запроса и списка релевантных чанков.
   - Формирование контекста на основе выбранных чанков.
   - Генерация ответа с использованием LLM.

### Сильные стороны

1. **Использование эмбеддингов для семантического поиска**:
   - Позволяет находить релевантную информацию независимо от формата и структуры текста.
   
2. **Диверсификация ответов с использованием RAG подхода**:
   - Учитывает контекст при генерации ответов, что позволяет использовать различные источники информации.

3. **Modular design**:
   - Разделение системы на отдельные компоненты (indexing, storage, search, ask2), что упрощает поддержку и масштабирование.

### Проблемы

1. **Простота индексации книг**:
   - Эмбеддинги для новых книг вычисляются каждый раз при индексации.
   - Отсутствует кэширование эмбеддингов, что увеличивает время обработки.

2. **Невозможность поиска в нескольких книгах**:
   - Функция `semantic_search` в данный момент работает только с одной книгой.

3. **Ограниченный контекст для ответа**:
   - Чанки извлекаются из начала книги, что ограничивает доступ к полному контексту.

4. **Дублирование кода**:
   - Функция `semantic_search_and_answer` используется в различных местах, но не всегда обновляется в соответствии с изменениями в остальной части системы.

### Баги или архитектурные риски

1. **Отсутствие тестирования на производительность**:
   - Не проводится масштабное тестирование для определения узких мест и оптимизации производительности.

2. **Проблемы с поддержкой новых форматов файлов**:
   - Текущая реализация поддерживает только `.txt` и `.epub`. Необходимо добавить поддержку других форматов, таких как PDF.

3. **Ограниченный контекст для ответа**:
   - Чанки извлекаются только из начала книги, что ограничивает доступ к полному контексту. Это может привести к неполным и недостаточными ответам.

### Качество RAG

- **Retrieval**:
  - Семантический поиск на основе эмбеддингов обеспечивает высокий уровень точности в поиске релевантных чанков.
  
- **Reranking**:
  - Реализовано эвристическое ранжирование результатов, что помогает в некоторых случаях улучшить качество ответа.

- **Answer generation**:
  - Использование LLM для генерации ответов на основе контекста.
  - Проблемы с точностью и полнотой ответов, особенно при наличии сложных или амбигуитетных запросов.
